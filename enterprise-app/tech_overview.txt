=========================================================
  สคริปต์อธิบายเทคนิค — HR System Enterprise
=========================================================

ภาพรวมสถาปัตยกรรม
---------------------------------------------------------
โปรเจกต์นี้ใช้รูปแบบ SPA (Single-Page Application)
แบ่งการทำงานออกเป็น 2 ฝั่ง:

  [ Vue.js (Browser) ]  <-->  [ Laravel API ]  <-->  [ TiDB Cloud ]
       Frontend                   Backend               Database


=========================================================

=========================================================

=========================================================
  ระบบ Authentication & Authorization
=========================================================

ขั้นตอน:
  Login
    --> ส่ง email + password ไป POST /api/login
    --> Laravel ตรวจด้วย Auth::attempt()
    --> ออก Sanctum Token ส่งกลับ
    --> Vue เก็บ token ใน localStorage
    --> ทุก request ถัดไปแนบ "Authorization: Bearer {token}"

2 ระดับ Middleware:
  1. auth:sanctum  --> ทุก API route ต้อง login ก่อน
  2. admin_hr      --> เฉพาะ HR Manager / Super Admin
                       ถึงจะสร้าง/แก้ไข/ลบข้อมูลหลักได้

=========================================================
  Flow การทำงานหลัก
=========================================================

[Login]
  Vue กรอก email+password
    --> POST /api/login
    --> Laravel ตรวจ Auth::attempt()
    --> สร้าง Token --> ส่งกลับ
    --> Vue เก็บ token + redirect Dashboard

[ดึงข้อมูลพนักงาน]
  Vue โหลดหน้า
    --> GET /api/employees (แนบ token)
    --> Middleware ตรวจ token
    --> Controller JOIN ตาราง users + branches + positions + types
    --> ส่งกลับ JSON ครบทุก field
    --> Vue แสดงในตาราง

[Token หมดอายุ]
  API ตอบ 401
    --> axios interceptor ดักจับ
    --> ลบ token + user จาก localStorage
    --> Redirect ไป /login อัตโนมัติ


=========================================================
  เทคนิคพิเศษในโปรเจกต์
=========================================================

1. Role-based Access Control (RBAC)
   ตรวจสอบจากชื่อตำแหน่งใน DB:
   allowedRoles = ['super admin', 'hr manager', 'system admin']

2. Temporary Position (รักษาการแทน)
   - บันทึก temp_position_end_date ไว้
   - Dashboard โหลดแล้วตรวจวันหมดอายุ
   - คืนตำแหน่งเดิมอัตโนมัติ

3. Navigation Guard
   ก่อนเข้าทุกหน้า router.js ตรวจ token + role
   ถ้าไม่มีสิทธิ์ --> redirect ไปหน้าที่เหมาะสม

4. DB::table JOIN แทน View
   View employees_list ขาดบาง field
   --> ใช้ DB::table JOIN โดยตรงให้ครบถ้วน
